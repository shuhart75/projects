# API AI Simulations Agent

Документация предназначена для аналитиков и разработчиков. Показывает, как взаимодействовать с AI-агентом RAIN для симуляций в продуктах банка.

---

## Методы API

### `POST /dialog/init` — Инициализация диалога с агентом

**Назначение:** Отправка контекста симуляции для запуска AI-агента. Это начало сессии, а не запуск самой симуляции.

**Что передаётся:**
- `sessionId` — идентификатор сессии (генерируется АС КОДА)
- `contextPrompt` — объект с параметрами симуляции (product, date, riskParam и др.)

**Ответ агента:**
- `agentResponse` — текстовое сообщение или запрос дополнительной информации
- `btUrl` (опционально) — ссылка на сгенерированный БТ

---

### `POST /dialog/{sessionId}/continue` — Продолжение диалога с агентом

**Назначение:** Отправка `userPrompt` (реакции пользователя) на запрос агента для продолжения диалога.

**Что передаётся:**
- `sessionId` — в URL (path-параметр)
- `userPrompt` — текст ответа пользователя

**Ответ агента:**
- `agentResponse` — текстовое сообщение или запрос дополнительной информации
- `btUrl` (опционально) — ссылка на сгенерированный или обновлённый БТ

---

## Схема взаимодействия

1. **Инициализация** — отправляете `POST /dialog/init` с параметрами симуляции
2. **Продолжение диалога** — если агент запросил уточнения, отправляете `POST /dialog/{sessionId}/continue` с ответом
3. **Повторение** — цикл продолжается до завершения симуляции

## Примеры запросов и ответов

### 1. Инициализация диалога (новая симуляция без БТ)

**Сценарий:** Первый запрос для инициализации диалога по кредитным картам, без существующего БТ.

**Запрос:**

```json
{
  "sessionId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "contextPrompt": {
    "simulationId": "sim-001-2024",
    "product": "Кредитные карты",
    "date": "2026-04-20",
    "name": "Изменение Риск-параметров",
    "user": "Иванов Иван Иванович",
    "businessEffect": "Снижение отказов на 5% за счет более мягких условий",
    "bt": null,
    "riskParam": [
      {
        "name": "MCOE",
        "asIs": 0.15,
        "toBe": 0.20
      },
      {
        "name": "Коэффициент одобрения",
        "asIs": 70,
        "toBe": 75
      },
      {
        "name": "Минимальный доход",
        "asIs": "30000",
        "toBe": "25000"
      },
      {
        "name": "Дата вступления",
        "asIs": "2025-12-01",
        "toBe": "2026-06-01"
      }
    ]
  }
}
```

**Ответ агента:**

> Я изучил параметры симуляции. Для продукта "Кредитные карты" изменены следующие риски:
>
> 1. MCOE: 0.15 → 0.20  
> 2. Коэффициент одобрения: 70 → 75  
> 3. Минимальный доход: 30000 → 25000  
> 4. Дата вступления: 2025-12-01 → 2026-06-01  
>
> Бизнес-эффект: Снижение отказов на 5% за счет более мягких условий.
>
> Для продолжения мне нужна дополнительная информация:
> - Какой тип симуляции вы хотите (проверка рисков / прогноз / аналитика)?
> - Есть ли ограничения по срокам реализации?

>
> **Изменяемые параметры:**
> - MCOE: 0.15 → 0.20 (+5 п.п.)
> - Коэффициент одобрения: 70 → 75 (+5 п.п.)
> - Минимальный доход: 30000 → 25000 (-5000)
> - Дата вступления: 2025-12-01 → 2026-06-01
>
> **Оценка рисков:**

> • Ожидаемый рост портфеля: +8%  
> • ROI: 28% за 6 месяцев
>
> БТ подготовлен. Согласны с формированием итогового документа?
>
> **Ссылка на БТ:** https://confluence.company.ru/pages/viewpage.action?pageId=123456

**Ответ агента:**

>

> 2. Ввести бонусы для держателей зарплатных карт  
> 3. Оптимизировать логистику выдачи наличных
>
> Для дальнейшей работы мне нужен фокус симуляции:
> - Оптимизация ставок?  
> - Изменение скоринг-политики?  
> - Новые продукты?
>
> Что вас интересует?

### 5. Инициализация диалога со строковыми и числовыми риск-параметрами

**Сценарий:** Пример с разными типами значений в `riskParam` — числа и строки.

**Запрос:**

  "sessionId": "c3d4e5f6-a7b8-9012-cdef-123456789012",
  "contextPrompt": {
    "simulationId": "sim-string-2024",
    "product": "Автокредиты",
    "date": "2026-04-20",
    "name": "Изменение условий автокредитования",
    "user": "Сидоров Сидор Сидорович",
    "businessEffect": "Увеличение объема автокредитов",
    "bt": null,
    "riskParam": [
      {
        "name": "Ставка для дилеров",
        "asIs": 9.5,
        "toBe": 8.9
      },
      {
        "name": "Срок кредита",
        "asIs": "До 7 лет",
        "toBe": "До 10 лет"
      },

        "name": "Первоначальный взнос",

**Ответ агента:**

>

> 4. Категория авто: Новые → Новые + подержанные
>
> Анализирую риски... Требуется уточнение:
> - Какие параметры подержанных авто рассматривать (возраст до 5 лет / без ограничений)?
> - Есть ли требования к пробегу?

## Ошибки валидации

### Пример 6: Отсутствует обязательное поле

**Запрос (ошибка):** без `sessionId`

```json
{
  "contextPrompt": {
    "date": "2026-04-20"
  }
}
```

**Ответ:**

```json
{
  "error": "ValidationError",
  "message": "Missing required fields: sessionId",
  "details": [
    {
      "field": "sessionId",
      "message": "sessionId is required"
    }
  ]
}
```

---

### Пример 7: Невалидное значение продукта

**Запрос (ошибка):** `product: "Ипотека"` (нет в enum)

```json
{
  "sessionId": "d4e5f6a7-b8c9-0123-defa-123456789012",
  "contextPrompt": {
    "simulationId": "sim-invalid",
    "product": "Ипотека",
    "date": "2026-04-20"
  }
}
```

**Ответ:**

```json
{
  "error": "ValidationError",
  "message": "Invalid value for product. Allowed: Кредитные карты, Потребительское кредитование, Автокредиты",
  "details": [
    {
      "field": "contextPrompt.product",
      "message": "Value must be one of: Кредитные карты, Потребительское кредитование, Автокредиты"
    }
  ]
}
```

## Памятка аналитика

| Поле | Тип | Обязательное | Примечание |
|------|-----|--------------|------------|
| `sessionId` | uuid | ✅ | Генерируется АС КОДА |
| `product` | string | ✅ | Только: КК, ПК, Автокредиты |
| `date` | date | ✅ | Дата запуска диалога |
| `name` | string | ❌ | Дефолт: "Изменение Риск-параметров" |
| `user` | string | ❌ | ФИО из СУДИРа |
| `businessEffect` | string | ❌ | Может быть null |
| `bt` | string/null | ❌ | URL БТ или null |
| `riskParam` | array | ❌ | Массив объектов с `name`, `asIs`, `toBe` |

**Типы значений в `riskParam`:**
- `asIs` и `toBe` могут быть: **число**, **строка**, **boolean**
- Примеры: `0.15`, `"30000"`, `"До 7 лет"`, `true`
